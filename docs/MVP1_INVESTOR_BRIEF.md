# ALIVE Connection MVP 1단계
## 투자자 기술 비전 브리프

---

## Executive Summary

**ALIVE Connection**은 NFC 기술을 활용하여 오프라인 만남의 순간을 디지털 자산으로 전환하는 **관계 지능 플랫폼**입니다. MVP 1단계에서는 "제로 입력 연락처 교환"이라는 핵심 가치를 구현하여, 사용자가 타이핑 없이 1초 만에 상대방의 연락처를 획득하고 만남의 맥락(시간, 장소)을 자동으로 기록하는 경험을 제공합니다.

---

## 1. 왜 이 문제를 해결해야 하는가

### 현재 네트워킹의 한계

비즈니스 네트워킹 현장에서 우리는 여전히 **1990년대의 방식**으로 인맥을 관리합니다. 컨퍼런스에서 만난 50명의 사람들, 그 중 실제로 연락처를 저장한 사람은 몇 명일까요? 종이 명함을 받아도 책상 서랍에 쌓이고, 수동으로 연락처를 입력하다 지쳐 포기합니다.

더 큰 문제는 **맥락의 상실**입니다. "이 사람을 어디서 만났지?", "무슨 이야기를 나눴더라?" 3개월 후 연락처 목록을 보면 이름만 있을 뿐, 그 만남의 가치는 사라져 있습니다.

### ALIVE가 해결하는 문제

ALIVE Connection은 이 문제를 근본적으로 해결합니다. **만남의 순간, 모든 것이 자동으로 기록됩니다.** 누구를, 언제, 어디서 만났는지. 사용자는 아무것도 입력하지 않습니다. 단지 폰을 맞대기만 하면 됩니다.

## 2. MVP 1단계의 기술적 핵심

### 2.1 NFC 기반 즉시 연결 (Zero-Input Exchange)

MVP 1단계의 핵심은 **NFC(Near Field Communication) 기술**을 활용한 즉시 연락처 교환입니다.

두 사용자가 스마트폰을 맞대는 순간, NDEF(NFC Data Exchange Format) 프로토콜을 통해 프로필 정보가 양방향으로 교환됩니다. 이 과정은 1초 이내에 완료되며, 사용자는 어떤 버튼도 누르지 않고, 어떤 텍스트도 입력하지 않습니다.

기존 QR코드 기반 솔루션과의 차별점은 **동시성**입니다. QR코드는 한 사람이 보여주고, 한 사람이 스캔하는 일방향 프로세스입니다. ALIVE의 NFC 핸드셰이크는 두 사람이 동시에 상대방의 정보를 받습니다. 이것은 단순한 기술적 차이가 아니라, "서로 연결되었다"는 심리적 대칭성을 만들어냅니다.

### 2.2 컨텍스트 자동 캡처 (Contextual Memory)

연락처 교환이 발생하는 순간, 시스템은 자동으로 **만남의 컨텍스트**를 기록합니다.

GPS 좌표를 획득하고, 역지오코딩(Reverse Geocoding)을 통해 "스타벅스 강남역점"과 같은 인간이 이해할 수 있는 장소명으로 변환합니다. 타임스탬프와 함께 저장된 이 정보는 나중에 "강남에서 만난 사람", "CES에서 만난 사람"과 같은 맥락 기반 검색을 가능하게 합니다.

이것이 ALIVE를 단순한 연락처 앱이 아닌 **관계 기억 시스템**으로 만드는 핵심입니다. 우리는 연락처를 저장하는 것이 아니라, 만남의 기억을 저장합니다.

### 2.3 듀얼 모드 아이덴티티 (Contextual Identity)

사람들은 상황에 따라 다른 정체성을 공유하고 싶어합니다. 비즈니스 미팅에서는 이메일과 LinkedIn을, 캐주얼한 모임에서는 Instagram과 WhatsApp을 공유하고 싶습니다.

ALIVE는 **Business 모드**와 **Casual 모드**를 제공합니다. 사용자는 상황에 맞는 모드를 선택하고, 해당 모드에 설정된 연락처만 상대방에게 공유됩니다. 이것은 프라이버시 보호이자, 상황에 맞는 적절한 정보 공개입니다.

홈 화면에서 한 번의 탭으로 모드를 전환할 수 있으며, NFC 교환 시 현재 활성화된 모드의 프로필만 전송됩니다.

### 2.4 시간 기반 관계 타임라인 (Temporal Relationship View)

수집된 연결은 **시간순 타임라인**으로 정렬됩니다. "오늘", "어제", "이번 주"와 같은 자연어 섹션으로 그룹핑되어, 사용자는 자신의 네트워킹 활동을 시간의 흐름 속에서 파악할 수 있습니다.

각 연결 카드에는 상대방의 기본 정보와 함께 만남 장소가 태그로 표시됩니다. 이것은 단순한 연락처 목록이 아니라, 내 네트워킹 여정의 기록입니다.

## 3. 기술 스택과 선택의 이유

### 3.1 React Native + Expo

크로스 플랫폼 개발을 위해 React Native와 Expo를 선택했습니다. 이를 통해 iOS와 Android를 동시에 지원하면서도 빠른 개발 속도를 유지합니다. Expo의 관리형 워크플로우는 NFC, GPS, 햅틱 피드백 등 네이티브 기능에 대한 접근을 단순화합니다.

### 3.2 Supabase (PostgreSQL + PostGIS)

백엔드로 Supabase를 선택한 이유는 세 가지입니다.

첫째, **PostgreSQL의 안정성**입니다. 관계형 데이터베이스는 사용자-연결-장소 간의 복잡한 관계를 표현하는 데 최적입니다.

둘째, **PostGIS 확장**입니다. 위치 기반 쿼리("강남에서 만난 모든 사람")를 효율적으로 처리하기 위해 공간 데이터베이스 기능이 필요합니다. PostGIS는 이를 네이티브로 지원합니다.

셋째, **실시간 동기화**입니다. Supabase의 실시간 구독 기능은 향후 메시징, 알림 등의 기능 확장에 필수적인 인프라입니다.

### 3.3 그래프 데이터 모델링

데이터베이스 스키마는 **노드-엣지 패턴**으로 설계되었습니다. 사용자(User)는 노드이고, 만남(Interaction)은 두 노드를 연결하는 엣지입니다. 이 설계는 현재 PostgreSQL에서 동작하지만, 향후 Neo4j와 같은 그래프 데이터베이스로의 마이그레이션을 염두에 두고 있습니다.

이 아키텍처는 MVP 2단계의 **관계 지식그래프** 기능의 기반이 됩니다. "A와 B를 모두 아는 사람은 누구인가?", "가장 자주 만나는 산업군은 무엇인가?"와 같은 그래프 쿼리를 가능하게 합니다.

## 4. MVP 1단계가 검증하는 것

### 4.1 핵심 가설

MVP 1단계는 다음 가설을 검증합니다:

**"사람들은 네트워킹 현장에서 더 빠르고 자동화된 연락처 교환 방식을 원하며, 만남의 맥락이 함께 기록되는 것에 가치를 느낀다."**

### 4.2 성공 지표

- **NFC 핸드셰이크 완료율**: 시도 대비 성공적으로 완료된 교환의 비율
- **재사용률**: 첫 교환 후 7일 내 추가 교환을 한 사용자 비율
- **타임라인 조회 빈도**: 사용자가 과거 연결을 얼마나 자주 되돌아보는가
- **검색 사용률**: 맥락 기반 검색("강남", "CES" 등)의 활용 빈도

### 4.3 학습 목표

- NFC 기반 교환의 실제 사용 패턴과 pain point
- 위치 정보의 정확도와 유용성
- 듀얼 모드의 실제 전환 빈도와 선호도

---

## 5. MVP 1단계 이후의 로드맵

MVP 1단계의 성공적인 검증 후, 다음 단계로 진화합니다:

### MVP 2단계: 관계 시각화

수집된 연결 데이터를 기반으로 **나를 중심으로 한 인맥 네트워크 그래프**를 시각화합니다. 관계의 강도(만남 횟수, 연락 빈도)에 따라 엣지의 두께가 달라지고, 회사/산업/장소별로 클러스터링된 인맥 지도를 제공합니다.

### MVP 3단계: 관계 지능

AI가 관계 패턴을 분석하여 인사이트를 제공합니다. "3개월간 연락하지 않은 중요 연결", "소개해드릴 만한 공통점이 있는 두 사람", "이번 주 네트워킹 요약 리포트" 등의 지능형 기능을 통해 ALIVE는 단순한 연락처 앱에서 **개인 관계 비서**로 진화합니다.

## 6. 경쟁 우위

| 기존 솔루션 | 한계 | ALIVE의 접근 |
|-------------|------|--------------|
| 종이 명함 | 수동 입력, 맥락 상실, 검색 불가 | 제로 입력, 자동 맥락 기록 |
| QR코드 명함앱 | 일방향 교환, 추가 앱 설치 필요 | 양방향 동시 교환 |

| CRM | 수동 입력 부담, 개인용 부적합 | 자동화된 개인 관계 관리 |

## 7. 결론

ALIVE Connection MVP 1단계는 **"만남을 기록하다"**라는 단순하지만 강력한 가치 제안을 기술적으로 구현합니다.

NFC를 통한 즉시 연결, GPS를 통한 자동 맥락 캡처, 그리고 시간 기반 타임라인은 함께 작동하여 사용자에게 완전히 새로운 네트워킹 경험을 제공합니다. 사용자는 더 이상 연락처를 "관리"하지 않습니다. 그저 만나고, 기억하고, 연결됩니다.

이것이 ALIVE가 만들어가는 **관계 지능의 첫 번째 장**입니다.
