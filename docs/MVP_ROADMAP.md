# ALIVE Connection - MVP 개발 로드맵

> **"기록은 최소화, 기억은 극대화"**
> NFC 기반 관계 지능 플랫폼

---

## 🎯 비전

**ALIVE Connection**은 단순한 명함 교환 앱이 아닙니다.
**나를 중심으로 한 인맥 지식그래프(Knowledge Graph)**를 자동으로 구축하여,
비즈니스 관계의 맥락과 가치를 시각화하고 인사이트를 제공하는 **관계 지능 플랫폼**입니다.

---

## 📊 지식그래프 온톨로지 설계

### 노드(Node) 정의
```
[Person] ─── 사용자 및 연결된 인맥
    ├── 기본 정보: 이름, 직함, 회사, 프로필 이미지
    ├── 소셜 링크: 이메일, 전화, LinkedIn, Instagram 등
    └── 메타데이터: 산업군, 관심사, 스킬

[Company] ─── 소속 회사/조직
    ├── 회사명, 산업군, 규모
    └── 위치 정보

[Location] ─── 만남 장소
    ├── 장소명, 주소, 도시, 국가
    └── GPS 좌표 (위도/경도)

[Event] ─── 만남 컨텍스트
    ├── 이벤트명 (예: "CES 2026", "스타트업 밋업")
    └── 날짜, 장소 연결
```

### 엣지(Edge/Relation) 정의
```
[MET_AT] ─── 만남 관계
    ├── 만난 시간 (timestamp)
    ├── 만난 장소 (Location 노드 연결)
    ├── 만남 컨텍스트 (Event 노드 연결)
    └── 메모, 음성메모

[WORKS_AT] ─── 소속 관계
    └── Person → Company

[CONTACTED] ─── 연락 이력
    ├── 연락 횟수
    ├── 마지막 연락 시간
    └── 연락 채널 (메시지, 전화 등)

[INTRODUCED_BY] ─── 소개 관계
    └── 누구를 통해 연결되었는지

[SIMILAR_TO] ─── 유사성 관계 (AI 추론)
    ├── 같은 회사
    ├── 같은 산업군
    ├── 같은 장소에서 만남
    └── 공통 관심사
```

## 📱 MVP 1단계: Core Connection (2월)
> **"만남을 기록하다"** - 핵심 NFC 연결 기능 완성

### 목표
- NFC로 즉시 연락처 교환
- 만남 정보 자동 저장 (위치, 시간)
- 기본 타임라인 조회

### 구현 기능

| 기능 | 설명 | 상태 |
|------|------|------|
| **NFC 핸드셰이크** | 폰을 맞대면 프로필 자동 교환 | ✅ 완료 |
| **Google 로그인** | Supabase Auth 기반 소셜 로그인 | ✅ 완료 |
| **프로필 등록** | 이름, 직함, 회사, 소셜 링크 입력 | ✅ 완료 |
| **위치 자동 캡처** | GPS + 역지오코딩으로 장소명 저장 | ✅ 완료 |
| **타임라인 뷰** | 날짜별 연결 히스토리 조회 | ✅ 완료 |
| **검색/필터** | 이름, 회사, 장소로 연결 검색 | ✅ 완료 |
| **듀얼 모드** | 비즈니스/캐주얼 프로필 전환 | ✅ 완료 |
| **Supabase 연동** | 실제 DB 저장 (Mock → 실제) | 🔧 진행중 |
| **프로필 상세** | 연결 상세 정보 및 메모 보기 | 🔧 진행중 |

### 데이터 모델 (1단계)
```sql
-- 핵심 노드
users (id, name, company, title, social_links, avatar_url)

-- 핵심 엣지
interactions (
  source_user_id,      -- 나
  target_user_id,      -- 만난 사람
  met_at,              -- 만난 시간
  location_*,          -- 만난 장소
  memo                 -- 메모
)
```

### 투자자 어필 포인트
```
✓ 제로 입력 연락처 교환 - 타이핑 없이 1초 만에 완료
✓ 자동 컨텍스트 저장 - 언제, 어디서 만났는지 자동 기록
✓ 기존 명함앱 대비 10배 빠른 UX
```

## 📈 MVP 2단계: Relationship Graph (3월)
> **"관계를 시각화하다"** - 지식그래프 기반 관계망 구축

### 목표
- 나를 중심으로 한 인맥 네트워크 시각화
- 관계 강도 및 연락 빈도 추적
- 연결 그룹핑 및 태그 시스템

### 구현 기능

| 기능 | 설명 | 가치 |
|------|------|------|
| **관계 그래프 뷰** | 나를 중심으로 한 인맥 네트워크 시각화 | 내 인맥의 구조를 한눈에 파악 |
| **관계 강도 표시** | 연락 빈도, 만남 횟수 기반 엣지 두께 | 중요한 관계 식별 |
| **인맥 클러스터링** | 회사별, 산업별, 장소별 자동 그룹핑 | 인맥 카테고리 자동 분류 |
| **태그 시스템** | 커스텀 태그로 연결 분류 | #투자자 #개발자 #CES2026 |
| **인앱 메시징** | 연결된 사람과 직접 대화 | 후속 커뮤니케이션 |
| **연락 이력 추적** | 메시지, 통화 횟수 기록 | 관계 유지 현황 파악 |
| **프로필 조회 알림** | 누가 내 프로필을 봤는지 알림 | 관심도 확인 |
| **음성 메모** | 만남 직후 음성으로 메모 기록 | 빠른 컨텍스트 저장 |

### 데이터 모델 (2단계 확장)
```sql
-- 연락 이력 엣지
messages (sender_id, receiver_id, content, created_at)

-- 관계 메타데이터
interactions.metadata → {
  "contact_count": 5,           -- 연락 횟수
  "relationship_strength": 0.8, -- 관계 강도 (0~1)
  "last_contact_at": "..."      -- 마지막 연락
}

-- 프로필 조회 로그
profile_views (viewer_id, viewed_id, viewed_at)
```

### 그래프 쿼리 예시
```sql
-- 나와 가장 많이 연락한 TOP 10
SELECT target_user_id, COUNT(*) as contact_count
FROM messages WHERE sender_id = :my_id
GROUP BY target_user_id
ORDER BY contact_count DESC LIMIT 10;

-- 같은 장소에서 만난 사람들 (클러스터)
SELECT * FROM interactions
WHERE location_city = 'Seoul'
AND source_user_id = :my_id;
```

### 투자자 어필 포인트
```
✓ 세계 최초 NFC 기반 관계 지식그래프
✓ 관계의 가치를 정량화 - 누구와 얼마나 소통하는지 시각화
✓ 자동 인맥 분류 - AI가 관계 패턴 분석
✓ CRM 수준의 관계 관리를 개인에게 제공
```

## 🧠 MVP 3단계: Relationship Intelligence (4월~)
> **"관계에서 인사이트를 얻다"** - AI 기반 관계 추천 및 분석

### 목표
- AI 기반 관계 인사이트 제공
- 네트워킹 추천 시스템
- 팀/조직 단위 협업 기능

### 구현 기능

| 기능 | 설명 | 가치 |
|------|------|------|
| **AI 관계 인사이트** | "이 분과 3개월째 연락이 없습니다" | 관계 유지 리마인더 |
| **소개 추천** | "A님과 B님을 연결해드릴까요?" | 공통점 기반 매칭 |
| **네트워킹 추천** | "이 행사에서 만날 만한 분들" | 위치 기반 추천 |
| **관계 요약 리포트** | 월간/분기별 네트워킹 분석 | 인맥 성장 추적 |
| **공유 인맥 탐색** | "A님과 공통으로 아는 분 3명" | 네트워크 중첩 분석 |
| **팀 네트워크** | 팀원들의 인맥을 통합 조회 | B2B 기능 |
| **명함 스캔 통합** | 기존 종이 명함 OCR 변환 | 기존 인맥 마이그레이션 |
| **캘린더 연동** | 미팅 일정과 연결 자동 매칭 | 컨텍스트 강화 |

### AI 추론 엔진
```python
# 관계 강도 계산 알고리즘
relationship_strength = (
    meeting_count * 0.3 +      # 만남 횟수
    message_count * 0.25 +     # 메시지 횟수
    recency_score * 0.25 +     # 최근성 (최근 연락일수록 높음)
    profile_view_count * 0.1 + # 프로필 조회
    shared_context * 0.1       # 공통 컨텍스트 (같은 회사, 이벤트)
)

# 연락 권장 알림
if days_since_last_contact > 90:
    notify("오랜만에 연락해보세요!")
```

### 데이터 모델 (3단계 확장)
```sql
-- AI 추론 결과 저장
relationship_insights (
  user_id,
  target_user_id,
  insight_type,        -- 'reconnect', 'introduce', 'similar'
  insight_data JSONB,  -- AI 추론 상세
  created_at
)

-- 소개 추천
introductions (
  introducer_id,       -- 소개하는 사람
  person_a_id,
  person_b_id,
  reason,              -- 추천 이유
  status               -- pending, accepted, declined
)
```

### 투자자 어필 포인트
```
✓ AI가 관계를 관리해주는 개인 비서
✓ "약한 연결"의 가치 극대화 - 잊힌 인맥 재발견
✓ B2B 확장 가능 - 영업팀, 투자팀 협업 도구
✓ 네트워킹 ROI 측정 가능
```
